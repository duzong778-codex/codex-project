# 防打扰规则审查结论

## 结论

老爹的判断是正确的：非技术项目负责人不应该被要求判断技术半成品。把报错、依赖选择、文件结构、测试失败、半成品截图交给老爹确认，会让项目停在最不该停的位置。

## 已补强的规则

- 禁止把技术半成品判断升级给老爹。
- Codex 必须自行完成诊断、修复、验证，最多自动循环 3 轮。
- 只有缺少业务素材、触及账号/资金/隐私/合规/不可回滚操作，或 3 轮后仍需要业务取舍时，才允许打断老爹。
- 提问时必须给出推荐默认选项，并说明不回复时的安全默认行为。

## 当前限制

`$review-run` 的真实 Claude/Codex 联动在本机未跑通，原因是本机配置里的 `claude` / `codex` 外部命令不可用，并伴随 Windows 编码报错。确定性模拟模式通过，说明流程结构可用；真实联动需要后续修复本机命令路径或 CLI 环境。

## 使用方式

后续任务可以直接写：

```text
按本项目 AGENTS.md 和 MEMORY.md 执行，保持防打扰、防混乱。任务是：……
```

即使不写这句，当前项目和全局模板也已默认包含该规则。
